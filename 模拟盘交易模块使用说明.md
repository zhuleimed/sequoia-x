# Sequoia-X 模拟盘交易模块

> 版本: v1.0 | 更新: 2026-07-02

将 LLM 推荐的股票投入 T+1 模拟盘交易，自动执行买入、每日估值、多因子评分卖出，全程微信推送报告。

---

## 一、架构总览

### 1.1 核心数据流

```
T日 18:10（管线启动）
  │
  ├─ Step 1: 数据同步（main.py --sync-only）
  │
  ├─ Step 2: 策略选股+LLM（main.py）
  │   ├─ 8 大策略选股
  │   ├─ LLM 多维度分析
  │   ├─ → 解析 LLM 推荐股票（RECOMMEND 行/最优关注/策略频率）
  │   └─ → 写入 sim_buy_signals 表（status='pending'）
  │
  ├─ Step 3: 模拟盘更新（main.py --sim-update）
  │   ├─ 1. 读取 pending 买入信号
  │   ├─ 2. 检查涨停/停牌/仓位上限 → 以开盘价执行买入
  │   ├─ 3. 更新所有持仓的当日估值（收盘价）
  │   ├─ 4. 逐只运行卖出规则评分
  │   ├─ 5. 触发卖出的 → 以收盘价卖出 → 推送单股报告
  │   └─ 6. 写入账户日结 + 推送组合日报
  │
  └─ 全管线汇总推送

T+1日 开盘 → 买入执行（T 日的信号）
T+N日 收盘 → 卖出判定 → 平仓报告
```

### 1.2 T+1 模型（同 019 ETF 模拟盘）

```
T日 20:00（LLM 推荐）
  → 产生"待执行买入信号"（写入数据库，今日不交易）
  → 信号状态: pending

T+1日 20:00（--sim-update）
  → 用 T+1 日开盘价执行昨日待执行买入信号
  → 用 T+1 日收盘价更新持仓估值 + 判定卖出
```

### 1.3 模块文件清单

| 文件 | 职责 |
|:----|:------|
| `sequoia_x/simulation/config.py` | **全部可调参数**（账户/成本/评分阈值） |
| `sequoia_x/simulation/models.py` | SQLite 表定义 + CRUD |
| `sequoia_x/simulation/signals.py` | LLM 推荐解析 + 信号管理 + **公开 API** |
| `sequoia_x/simulation/engine.py` | **SimEngine** 核心引擎 |
| `sequoia_x/simulation/rules.py` | 多因子卖出评分引擎 |
| `sequoia_x/simulation/reporter.py` | 单股平仓报告 + 组合日报推送 |

## 二、配置参数

全部参数集中在 `sequoia_x/simulation/config.py`，直接修改即可生效。

### 2.1 账户参数

```python
INITIAL_CAPITAL   = 1_000_000   # 初始总资金 100 万
PER_STOCK_BUDGET  = 50_000      # 单只股票分配 5 万
MAX_POSITIONS     = 20           # 最大同时持仓数
```

### 2.2 交易成本（A 股）

```python
COMMISSION_RATE   = 0.00025     # 佣金 万2.5（双向）
STAMP_TAX_RATE    = 0.001       # 印花税 千1（卖出）
SLIPPAGE          = 0.0001      # 滑点 万1
```

### 2.3 卖出评分规则

| 规则 | 条件 | 分值 | 类型 |
|:----|:------|:----:|:----|
| S1 硬止损 | 亏损 > 8% | **100** | 穿透强制卖出 |
| S2 止损预警 | 亏损 > 5% | 40 | 累加评分 |
| T1 移动止盈强卖 | 从最高回落 ≥ 8%（须先盈利≥15%） | **85** | 累加评分 |
| T2 移动止盈预警 | 从最高回落 ≥ 5%（须先盈利≥15%） | 50 | 累加评分 |
| D1 时间止损 | 持有 > 20 日 | **75** | 累加评分 |
| D2 时间预警 | 持有 > 15 日 | 40 | 累加评分 |
| M 均线死叉 | MA5<MA10 连续 3 日 | **70** | 累加评分 |
| M 均线死叉初现 | MA5<MA10 单日 | 40 | 累加评分 |
| SH 夏普恶化 | 15 日夏普 < -0.5 | **70** | 累加评分 |
| SH 夏普转负 | 10 日夏普 < 0 | 50 | 累加评分 |
| SH 夏普偏低 | 15 日夏普 < 0.5 | 30 | 累加评分 |
| R1 相对弱势 | 近 5 日跑输指数 > 5% | **60** | 累加评分 |
| R2 相对弱势预警 | 近 5 日跑输指数 > 3% | 30 | 累加评分 |

**触发条件**：总分 ≥ **60** 或 S1 硬止损穿透。

## 三、新策略接入方式

### 3.1 方式一：纯 LLM 推荐（零改动）

新策略加入 `main.py` 的 `strategies` 列表 → LLM 自动分析 → 推荐股票自动进入模拟盘。

```python
# main.py
strategies: list[BaseStrategy] = [
    MaVolumeStrategy(...),         # 已有
    RpsBreakoutStrategy(...),      # 已有
    MyNewStrategy(...),            # 你的新策略 ← 加在这里即可
]
```

### 3.2 方式二：策略直接提交信号（推荐，一行代码）

```python
from sequoia_x.simulation.signals import submit_buy_signals

class MyNewStrategy(BaseStrategy):
    webhook_key = "my_strategy"
    display_name = "我的新策略"

    def run(self) -> list[str]:
        # ... 你的选股逻辑 ...
        selected = ["600519", "000858"]

        # ── 一行提交到模拟盘 ──
        submit_buy_signals(
            db_path=self.settings.db_path,
            symbols=selected,
            strategy_name=self.display_name,
            top_n=2,
        )
        return selected
```

### 3.3 方式三：直接写库（完全控制）

```python
from sequoia_x.simulation.models import insert_buy_signals_batch

insert_buy_signals_batch(settings.db_path, [
    {"symbol": "600519", "strategy_from": "手动", "llm_score": None},
])
```

### 3.4 自定义报告查看

```bash
# 查看某只股票的历史模拟交易记录
python main.py --sim-report 600519
```

## 四、每日运行流程

### 4.1 交易日流程（cron 18:10 触发）

```
1. 非交易日判断 → 跳过管线（含模拟盘）
2. 交易日：
   Step 1: 数据同步（增量拉取 T 日行情）
   Step 2: 策略选股+LLM（产生推荐 → 写入信号表）
   Step 3: 模拟盘更新（执行信号 → 估值 → 卖出判定 → 推送）
```

### 4.2 模拟盘更新内部逻辑

```python
# SimEngine.run_daily() 顺序
1. 检查交易日 + 数据就绪
2. 查询 sim_buy_signals WHERE status='pending'
3. 对每只信号：
   ├─ 涨停检查 → 涨停则取消
   ├─ 停牌检查 → 停牌则取消
   ├─ 仓位已达上限 → 取消
   ├─ 资金不足 → 取消
   └─ 全部通过 → 以开盘价买入 → 写入 sim_positions
4. 对每只持仓：
   ├─ 读取今日收盘价 → 更新估值（pnl/pnl_pct/highest_price）
   ├─ 运行 evaluate_exit() 多因子评分
   └─ 评分 ≥ 60 → 以收盘价卖出 → 移入 sim_closed_trades
5. 写入 sim_account_daily 账户总览
6. 推送组合日报到微信
```

### 4.3 涨停/停牌处理（同 019）

- **涨停**：开盘价 ≥ 前收盘 × 1.10 → 不能买入 → 信号取消（不顺延）
- **停牌**：当日成交量 = 0 → 不能买入 → 信号取消（不顺延）
- **T+1 保护**：今日买入的股票，今日不触发任何卖出检查
- **跌停**：开盘价 ≤ 前收盘 × 0.90 → 不能买入 → 信号取消

## 五、数据库表

### 5.1 表结构

| 表名 | 用途 | 关键字段 |
|:----|:-----|:---------|
| `sim_buy_signals` | 买入信号 | symbol, status(pending/executed/cancelled), buy_date |
| `sim_positions` | 当前持仓 | symbol, shares, buy_price, total_cost, highest_price, pnl_pct |
| `sim_closed_trades` | 已完成交易 | symbol, buy/sell_date, pnl, pnl_pct, sharpe_ratio, exit_reason |
| `sim_account_daily` | 账户日结 | date, cash, stock_value, total_value, total_return |

### 5.2 查看持仓

```sql
SELECT symbol, shares, buy_price, current_price, pnl_pct, hold_days
FROM sim_positions ORDER BY pnl_pct DESC;
```

### 5.3 查看交易记录

```sql
SELECT symbol, buy_date, sell_date, hold_days, pnl_pct, exit_reason
FROM sim_closed_trades ORDER BY sell_date DESC;
```

## 七、参数调优指引

### 7.1 常见调整场景

| 需求 | 修改参数 |
|:-----|:---------|
| 想让持仓拿久一点 | 提高 `TRAILING_T1`（从 8% 提到 10%）或 `MAX_HOLD_DAYS`（从 20 提到 30） |
| 想让卖出更敏感 | 降低 `SELL_THRESHOLD`（从 60 降到 50）或降低规则阈值 |
| 提高单只仓位上限 | 提高 `PER_STOCK_BUDGET`（从 5 万提到 8 万） |
| 扩大持仓规模 | 提高 `INITIAL_CAPITAL` 和 `MAX_POSITIONS` |
| 降低交易成本影响 | 降低 `COMMISSION_RATE`（A 股部分券商可到万 1.5） |

### 7.2 回测建议

当前模拟盘是**实盘延时模式**（T 日推荐 → T+1 日执行），与真实交易体验一致。建议：

1. **运行至少 1~2 个月**积累足够的交易样本
2. 重点关注：**胜率**、**盈亏比**、**夏普率**、**最大回撤**
3. 根据实际数据调整卖出评分阈值

## 八、与 019 ETF 模拟盘的差异对比

| 维度 | 019 ETF 模拟盘 | 本模块（A 股个股） |
|:----|:-------------|:------------------|
| 标的类型 | ETF | A 股个股 |
| 持仓模型 | 单只全仓 | 多只等额（5万/只，限20只） |
| 信号来源 | 技术指标自动计算 | LLM 报告解析 / 策略直接提交 |
| 交易成本 | 佣金万2，免印花税 | 佣金万2.5 + 印花税千1（卖出） |
| 涨跌停限 | 10% / 20%（ETF 特殊） | 10%（A 股主板） |
| 状态存储 | 独立 JSON 文件 | SQLite 集中存储 |
| 卖出触发 | 风控+信号切换 | 多因子评分 ≥ 60 分 |
