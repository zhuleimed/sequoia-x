# Sequoia-X 模拟盘交易逻辑与运行全流程说明

> 版本: v2.0 | 最后更新: 2026-07-06
> 
> 本文档完整记录模拟盘从荐股到买卖、持股、换股、账户核算的全套逻辑与流程。
> 目标是让不熟悉代码的人也能完整理解模拟盘的运行机制。

---

## 一、核心原则

| # | 原则 | 说明 |
|:--|:-----|:------|
| 1 | **T+1 模型** | 所有买卖决策基于 T-1 日收盘价数据，在 T 日以开盘价执行。决策和执行隔一个交易日 |
| 2 | **先卖后买** | 每日执行顺序：先卖出（释放仓位）→ 再买入（递补） |
| 3 | **取消不顺延** | 涨跌停/停牌/仓位不足/资金不够 → 取消信号，不留存过夜。次日由 LLM 重新推荐（自然恢复）|
| 4 | **LLM 覆盖三次上限** | 卖出规则触发时若 LLM 同日推荐该股，可覆盖 ≤ 3 次。第 3 次仍触发则强制执行卖出 |
| 5 | **评分优先** | 买入信号按 LLM 评分降序排列。仓位不够时只取高分，低分直接丢弃 |
| 6 | **当日买入不可卖** | `today_opened=1` 标记保护，当日估值时跳过卖出评分。次日重置后可正常评分 |
| 7 | **先核算再执行** | 每个交易日先重置 T+1 标记并递增持有天数，再执行买卖，最后估值评估 |

## 二、时间线模型

### 2.1 交易日流程（管线步骤）

```
          T-1 日                              T 日                          T+1 日
     ═══════════════                    ═══════════                    ═══════════════

     09:30-15:00 盘中交易              09:30-15:00 盘中交易           09:30-15:00 盘中交易
     15:00 CLOSE 收盘                  15:00 CLOSE 收盘               15:00 CLOSE 收盘

18:10 cron 触发管线                  18:10 cron 触发管线              18:10 cron 触发管线
  │                                     │                                │
  ├─ Step1: 数据同步                    ├─ Step1: 数据同步                ├─ Step1: 数据同步
  │  拉取 T-1 日行情入库                │  拉取 T 日行情入库              │  拉取 T+1 日行情入库
  │  股票列表/日线/指数                 │  股票列表/日线/指数              │  股票列表/日线/指数
  │                                     │                                │
  ├─ Step2: 策略选股+LLM                ├─ Step2: 策略选股+LLM           ├─ Step2: 策略选股+LLM
  │  8策略选股 → LLM 分析               │  8策略选股 → LLM 分析           │  8策略选股 → LLM 分析
  │  → 写入 pending 买入信号            │  → 写入 pending 买入信号         │  → 写入 pending 买入信号
  │  （T-1日推荐，T日执行）            │  （T日推荐，T+1日执行）         │  （T+1日推荐，T+2日执行）
  │                                     │                                │
  └─ Step3: --sim-update                └─ Step3: --sim-update           └─ Step3: --sim-update
      ├─ ① 重置today_opened=0              ├─ ① 重置today_opened=0          ├─ ① 重置today_opened=0
      │     hold_days+=1                   │     hold_days+=1               │     hold_days+=1
      ├─ ② 执行待卖出(OPEN价)              ├─ ② 执行待卖出(OPEN价)          ├─ ② 执行待卖出(OPEN价)
      ├─ ③ 执行待买入(OPEN价)              ├─ ③ 执行待买入(OPEN价)          ├─ ③ 执行待买入(OPEN价)
      └─ ④ 估值+卖出评估(CLOSE价)          └─ ④ 估值+卖出评估(CLOSE价)      └─ ④ 估值+卖出评估(CLOSE价)
```

### 2.2 关键时序术语

```
T-1日 18:10 评估持仓（用 T-1 日 CLOSE）
  → 触发卖出条件 → 标记 pending_sell（不实际卖出）
  → LLM 推荐 → 写入 pending 买入信号

T日 18:10 执行待卖出（用 T 日 OPEN）
  → pending_sell 标记的持仓以 OPEN 卖出 → 释放仓位
  → 再用 T 日 OPEN 买入 T-1 日 LLM 推荐的股票 → 递补
  → 评估持仓（用 T 日 CLOSE）→ 标记新的 pending_sell
  → LLM 推荐 → 写入新的 pending 买入信号

T+1 日重复上述流程
```

| 术语 | 含义 |
|:----|:------|
| **CLOSE 价** | 用于评估（多因子评分），决定是否触发买卖条件 |
| **OPEN 价** | 用于执行（实际买卖操作），决定成交价格 |
| **T-1 日推荐** | 上一个交易日 LLM 的推荐。今天执行买入 |
| **T 日推荐** | 今天 LLM 的推荐，写入 pending 信号。明天执行买入 |
| **pending 信号** | `sim_buy_signals` 表中 `status='pending'` 的记录，待次日执行 |
| **pending_sell** | `sim_positions` 表中 `pending_sell_reason` 非空的持仓，待次日开盘卖出 |

## 三、买入逻辑

### 3.1 信号来源（3 种方式）

```
LLM 报告 → parse_llm_report()
  ├─ 策略1：解析 "RECOMMEND: 600519,000858" 末行（最精确，由prompt指定格式）
  ├─ 策略2：解析 "最优关注: 宁德时代(300750)" 行中的股票代码
  └─ 策略3（回退）：按多策略选中频率取前 2 只

公开 API → submit_buy_signals(db_path, symbols, strategy_name)
  任意策略 run() 中一行代码直接提交

直接写库 → insert_buy_signals_batch(db_path, [{"symbol":"...", ...}])
  最底层方式，完全控制
```

LLM 每次推荐 **0~2 只**（可推荐 0 只、1 只或 2 只）。如果 LLM 推荐 0 只（表示当日无推荐），则**不回退**到策略频率，尊重 LLM 的判断。

### 3.2 信号生命周期

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  pending  │ ──▶  │ executed  │      │ cancelled │
│ (待执行)  │      │ (已买入)  │      │ (已取消)  │
└──────────┘      └──────────┘      └──────────┘
       │                                  ▲
       │  buy_date >= today               │
       │  → 明天才处理                     │ 取消原因：
       │                                  │ 涨停/停牌/资金不足/
       └──────────────────────────────────┤ 仓位已满
                                          │
                                  保留到次日？❌ 不留存
```

### 3.3 信号写入 → 排序 → 执行

```
T-1日 Step2（LLM 推荐，18:10）
  → insert_buy_signals() → buy_date = T-1, status = 'pending'
  ↓
T-1日 Step3（--sim-update，18:10）
  → get_pending_signals() WHERE buy_date < 'T-1'
  → 取不到 T-1 日信号（因为 buy_date = T-1，不满足 < T-1）
  → 不做处理 ✓ 等待明天
  ↓
T日 Step3（--sim-update，18:10）
  → get_pending_signals() WHERE buy_date < 'T'
  → 取到 T-1 日推荐的信号（buy_date = T-1 < T）
  → 按 llm_score DESC 排序
  ↓
  计算可用仓位：slots_available = 20 - 当前持仓数
  ↓
  signals[:slots_available]  → 执行买入 ✅
  signals[slots_available:]  → mark_signal_cancelled() ❌
```

**关键 SQL 逻辑**（`get_pending_signals`）：
```sql
SELECT ... FROM sim_buy_signals
WHERE status = 'pending' AND buy_date < ?
ORDER BY llm_score IS NOT NULL DESC, llm_score DESC, buy_date ASC, id ASC
```

- `buy_date < ?`：严格小于今天，确保**今天写入的信号明天才处理**
- `llm_score IS NOT NULL DESC`：有评分的排前面
- `llm_score DESC`：评分高的排前面
- `buy_date ASC, id ASC`：同评分则先写入的先处理

### 3.4 买入检查清单

每个信号在执行之前，依次检查下列条件。**任一条件不通过 → 信号取消，不重试**：

| # | 检查项 | 条件 | 失败处理 |
|:--|:-------|:-----|:---------|
| 1 | 行情数据 | 今日 `stock_daily` 表中有该股票的行情记录 | `cancelled` |
| 2 | 涨停 | `open_price < prev_close × 1.10` | `cancelled` |
| 3 | 跌停 | `open_price > prev_close × 0.90` | `cancelled` |
| 4 | 停牌 | `volume > 0`（今日有成交量） | `cancelled` |
| 5 | 资金充足 | `预算5万 ≥ 买入1手的金额` | `cancelled` |
| 6 | 现金足够 | `现金余额 ≥ 买入总成本` | `cancelled` |
| 7 | 仓位未满 | `当前持仓数 < 20` | `cancelled` |

**特别说明**：第 5 项是常见取消原因——高价股（如贵州茅台约 1500 元/股），1 手（100 股）需要 15 万元，超出单只预算 5 万，自动取消。

### 3.5 买入价格与成本计算

```
买入价 = OPEN价 × (1 + SLIPPAGE)
SLIPPAGE = 万1 = 0.0001         ← 模拟交易滑点

预算 = min(PER_STOCK_BUDGET(5万), 当前现金余额)
股数 = floor(预算 / 买入价 / 100) × 100  ← 向下取整到整手(100股)

成本 = 股数 × 买入价
佣金 = max(成本 × 万2.5, 0)
总成本 = 成本 + 佣金            ← 从现金中扣除的总额
```

### 3.6 仓位不足处理

```
持仓 19 只，LLM 推荐 2 支（评分 4.5 和 4.0）
  ↓
slots_available = 20 - 19 = 1
signals[:1] → 评分 4.5 的执行买入 ✅
signals[1:] → 评分 4.0 的取消丢弃 ❌（不保留到下一日）
  ↓
下一交易日 LLM 基于最新行情重新推荐
  → 可能再次推荐该股，也可能推荐别的
  → 用新信号递补，不用旧信号
```

### 3.7 日报中的买入信息

日报中 "今日买入" 区块显示成功买入的股票，"未执行（取消原因）"区块显示被取消的信号及原因：

```
  ▶ 今日买入
    ✅ 五粮液(000858) 600股 @ 72.92

  ▶ 未执行（取消原因）
    ⏭️ 600519 → 资金不足（预算50000，不够1手）
```

## 四、卖出逻辑

### 4.1 卖出四阶段流程

```
┌──────────────────────────────────────────────────────────────┐
│  阶段一：估值更新（T 日 CLOSE 价）                             │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │ 遍历所有持仓，对每只：                                    │ │
│  │  1. 从 stock_daily 取 T 日 CLOSE 价                     │ │
│  │  2. update_position_valuation()                         │ │
│  │     ├─ 更新 current_price, current_value, pnl, pnl_pct  │ │
│  │     ├─ 更新 highest_price（创新高则更新）                 │ │
│  │     └─ 不变 hold_days, today_opened                     │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│  阶段二：多因子评分（T 日 CLOSE 价）                           │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │ evaluate_exit(entry, current, highest, hold_days, ...)  │ │
│  │   ├─ 13 条规则各自独立评分                                │ │
│  │   ├─ 评分累加 = 总分                                     │ │
│  │   ├─ S1 硬止损（100分）= 穿透，不经总分判断直接卖出       │ │
│  │   └─ 其他规则累加后判断总分 ≥ 60                         │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│  阶段三：决策（规则引擎 + LLM 覆盖）                           │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │  总分 ≥ 60 或 S1 穿透？                                  │ │
│  │    ├─ 否 → 继续持有 ✅                                   │ │
│  │    └─ 是 → LLM 同日推荐该股？                            │ │
│  │       ├─ 是 → 检查覆盖次数：                              │ │
│  │       │   < 3次 → 跳过卖出，覆盖计数+1，继续持有 ⚠️      │ │
│  │       │   ≥ 3次 → 强制执行卖出 🔴                         │ │
│  │       └─ 否 → mark_position_for_sell() ⚠️                │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                              ↓
┌──────────────────────────────────────────────────────────────┐
│  阶段四：执行卖出（T+1 日 OPEN 价）                            │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │ _execute_pending_sells()                                │ │
│  │   ├─ 查 pending_sell_reason IS NOT NULL 的持仓           │ │
│  │   ├─ 检查跌停：open ≤ prev_close×0.90 → 取消标记 ⏭️     │ │
│  │   ├─ 检查停牌：volume=0 → 取消标记 ⏭️                    │ │
│  │   └─ 通过 → 以 OPEN 价执行卖出 ✅                        │ │
│  │       ├─ 计算：滑点/佣金/印花税/PnL                      │ │
│  │       ├─ 写入 sim_closed_trades                         │ │
│  │       ├─ 从 sim_positions 删除                          │ │
│  │       └─ 推送单股平仓报告到微信                          │ │
│  └─────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 多因子评分规则（13 条）

| 编号 | 规则名称 | 触发条件 | 分值 | 穿透？ | 说明 |
|:----:|:---------|:---------|:----:|:------:|:-----|
| S1 | **硬止损** | 亏损 > 8% | **100** | ✅ | 不限总分，直接卖出 |
| S2 | 止损预警 | 亏损 > 5% | 40 | | 累加至总分 |
| T1 | **移动止盈强卖** | 从最高点回落 ≥ 8%（须先盈利 ≥ 15%） | **85** | | 已盈利股票回吐利润 |
| T2 | 移动止盈预警 | 从最高点回落 ≥ 5%（须先盈利 ≥ 15%） | 50 | | 预警级别 |
| D1 | **时间止损** | 持有 > 20 个交易日 | **75** | | 防长期套牢 |
| D2 | 时间预警 | 持有 > 15 个交易日 | 40 | | 接近上限 |
| M1 | **均线死叉确认** | MA5 < MA10 连续 ≥ 3 日 | **70** | | 趋势已确认转弱 |
| M2 | 均线死叉初现 | MA5 < MA10 单日 | 40 | | 首次出现 |
| SH1 | **夏普恶化** | 15 日滚动夏普率 < -0.5 | **70** | | 风险调整后收益很差 |
| SH2 | 夏普转负 | 10 日滚动夏普率 < 0 | 50 | | 近期表现不佳 |
| SH3 | 夏普偏低 | 15 日滚动夏普率 < 0.5 | 30 | | 性价比偏低 |
| R1 | **相对弱势** | 近 5 日跑输上证指数 > 5% | **60** | | 相对强度不足 |
| R2 | 相对弱势预警 | 近 5 日跑输上证指数 > 3% | 30 | | 相对强度偏弱 |

**触发条件**：总分 **≥ 60**，或 S1 硬止损穿透（**100 分，不经总分判断**）。

### 4.3 移动止盈的激活逻辑

移动止盈不是任何时候都生效的，有严格的前提条件：

```
持仓收益率 < 15%？   → 移动止盈不激活（不检查回落）
持仓收益率 ≥ 15%？  → 移动止盈激活
                        ├─ 从最高回落 < 5% → 正常持有
                        ├─ 从最高回落 ≥ 5% → T2 预警（+50分）
                        └─ 从最高回落 ≥ 8% → T1 强卖（+85分）

示例：
  买入 100元 → 最高涨到 120元（+20%，已激活）
    → 回落至 114元（从120回落5%）= T2预警
    → 回落至 110.4元（从120回落8%）= T1触发（85分≥60 → 卖出）
    → 锁定 +10.4% 利润 ✅
```

### 4.4 卖出价格与成本计算

```
卖出价 = OPEN价 × (1 - SLIPPAGE)
SLIPPAGE = 万1 = 0.0001

收入 = 股数 × 卖出价
佣金 = max(收入 × 万2.5, 0)        ← 双向收取（买入已收一次）
印花税 = 收入 × 千1                 ← 仅卖出时收取（A股规定）

净收入 = 收入 - 佣金 - 印花税
盈亏 = 净收入 - 持仓总成本
收益率 = 盈亏 / 持仓总成本
```

### 4.5 T+1 保护机制（today_opened）

| 场景 | today_opened | 卖出评分行为 |
|:-----|:------------:|:------------|
| T 日刚买入，T 日评估 | **1** | 跳过（今日不可卖）✅ |
| T+1 日评估 | **0** | 正常评分，可触发卖出 |
| T+N 日评估 | **0** | 正常评分 |

**重置机制**：`_execute_pending_orders()` 每日开头执行：
```sql
UPDATE sim_positions SET today_opened=0  -- 所有持仓今日可卖
```
新买入的持仓通过 `insert_position()` 设 `today_opened=1`。所以：
- 今日新买入 → 保护 ✅（今天不能卖）
- 昨日买入 → 已重置为 0 → 正常评分

### 4.6 卖出失败处理（取消不顺延）

```
跌停（open ≤ prev_close × 0.90）→ clear_pending_sell()
  → 取消标记，不留存
  → 今晚 _update_and_evaluate() 用当日 CLOSE 重新评分
  → 若再次触发 → 重新标记 pending_sell → 明日再试
  → 若已不触发 → 继续持有（自然恢复）

停牌（volume = 0）→ clear_pending_sell()
  → 同上
```

**为什么取消而不顺延**：A 股个股很少连续跌停，今天的卖出理由（技术面恶化）到后天可能已变化。今晚重新评估比硬性顺延旧订单更合理。

### 4.7 日报中的卖出信息

日报中会显示两处卖出相关信息：

```
  ▶ 今日卖出          ← 当天已执行的卖出（实际平仓）
    🔄 浩物股份(000757) → +1,777.88 (+3.89%)
       原因: 移动止盈触发...

  ▶ 明日待卖出        ← 当天标记的 pending_sell（明天执行）
    ⚠️ 000757 → SH(夏普率)(70分); M(均线死叉)(40分)
```

## 五、LLM 覆盖机制

### 5.1 覆盖条件与流程

卖出规则触发时，如果同日 LLM 也推荐了该股票（`sim_buy_signals` 中有该股今天的 pending 信号），则 LLM 的推荐可以**覆盖**卖出规则，让股票继续持有。

```
卖出触发（评分 ≥ 60）
  └─ 该股在今日 LLM 推荐中？→ check sim_buy_signals WHERE buy_date=today
       ├─ 是 → LLM 覆盖生效
       │   ├─ 覆盖次数 < 3 → 跳过卖出，llm_override_count += 1
       │   └─ 覆盖次数 ≥ 3 → 不再覆盖，强制执行卖出
       └─ 否 → 正常标记 pending_sell
```

### 5.2 计数器机制

`sim_positions` 表中的 `llm_override_count` 字段记录该持仓已被 LLM 覆盖跳过卖出的次数：

```
初始值：0（insert_position 时设置）

每次覆盖：UPDATE sim_positions SET llm_override_count = llm_override_count + 1

覆盖次数 < 3  → 跳过卖出，继续持有 ⚠️
覆盖次数 ≥ 3  → 不再覆盖，强制 mark_position_for_sell() 🔴
                 退出原因 = "LLM覆盖3次后强制卖出(...)"

计数器重置：
  - 正常标记 pending_sell（未被覆盖）→ reset_llm_override() → 0
  - 卖出后持仓删除 → 计数器自动废弃
```

### 5.3 设计理由

LLM 做全维度分析（大盘环境、板块情绪、个股基本面、新闻公告），而技术规则只看价格指标（均线、夏普、回撤等）。LLM 说"买"可能看到了技术指标无法捕捉的利好。

但连续 3 次评分都说"卖"仍被覆盖，说明趋势确实恶化，不应再扛，第 3 次强制执行卖出。

## 六、每日执行顺序

### 6.1 `run_daily()` 三步流程

```
run_daily()
  │
  ├─ 第1步：_execute_pending_orders()  ← 执行买卖（用今日 OPEN 价）
  │   ├─ ① 重置 today_opened=0（昨日的T+1保护到期）
  │   ├─ ② hold_days+=1 所有持仓（每过一个交易日+1）
  │   ├─ ③ _execute_pending_sells()
  │   │   └─ 处理 pending_sell 标记的持仓 → 以 OPEN 价卖出
  │   └─ ④ _execute_pending_buys()
  │       └─ 处理 pending 买入信号 → 以 OPEN 价买入，设 today_opened=1
  │
  ├─ 第2步：_update_and_evaluate()  ← 估值 + 评估（用今日 CLOSE 价）
  │   ├─ ① 遍历所有持仓
  │   ├─ ② 读取今日 CLOSE 价 → 更新估值（市值/盈亏）
  │   ├─ ③ evaluate_exit() 多因子评分
  │   ├─ ④ 评分 ≥ 60？→ mark_position_for_sell()
  │   └─ ⑤ 已标记 pending_sell 或 today_opened=1 的跳过
  │
  ├─ 第3步：_write_account_daily()  ← 写入日结
  │   ├─ 现金 = 初始资金 - 持仓总成本 + 已平仓盈亏
  │   ├─ 持仓市值 = 遍历所有持仓的 current_value 之和
  │   └─ 总资产 = 现金 + 持仓市值
  │
  └─ 第4步：_push_daily_summary()  ← 推送日报
      └─ 账户概况 + 今日操作 + 明日待卖出 + 当前持仓
```

### 6.2 先卖后买的仓位递补示例

```
T-1 日收盘：持仓 20 只（满仓）
  LLM 推荐 2 支（评分 4.5 和 4.0）→ 写入 pending 信号
  1 只触发卖出条件 → 标记 pending_sell

T 日 --sim-update：
  Step 1-①：重置 today_opened=0，hold_days+=1
  Step 1-②：_execute_pending_sells()
    → 取出 pending_sell 的 1 只持仓
    → 以 OPEN 价卖出 → 释放 1 个仓位 → 持仓 19 只
  Step 1-③：_execute_pending_buys()
    → 读取 pending 信号（排序后：[评分4.5, 评分4.0]）
    → slots_available = 20 - 19 = 1
    → signals[:1] → 评分 4.5 的以 OPEN 价买入 ✅
    → signals[1:] → 评分 4.0 的取消 ❌
  Step 2：估值 + 评估 → 可能又触发新的卖出
  Step 3：日结 → 现金/市值/总资产
```

## 七、账户核算

### 7.1 现金计算公式

```python
现金 = INITIAL_CAPITAL - sum(sim_positions.total_cost) + sum(sim_closed_trades.pnl)
     = 1,000,000     - 持仓总成本                   + 已平仓盈亏
```

**分步说明**：
| 项目 | 公式 | 示例 |
|:-----|:-----|:------|
| 初始资金 | 1,000,000 | 1,000,000.00 |
| − 当前持仓成本 | 所有未卖出的股票买入总成本 | − 49,940(000757) − 43,761(000858) |
| + 已平仓盈亏 | 所有已卖出股票的盈亏之和 | + 0（尚无交易完成） |
| = **现金** | | **906,298.22** |

### 7.2 总资产计算公式

```
持仓市值 = Σ(sim_positions.shares × current_price)
总资产   = 现金 + 持仓市值
累计收益 = (总资产 / 1,000,000) − 1
```

### 7.3 持有天数（hold_days）计算

```
买入时：insert_position() 设置 hold_days=1
每日递增：_execute_pending_orders() 开头执行
    UPDATE sim_positions SET hold_days = hold_days + 1
估值时：update_position_valuation() 不改变 hold_days

示例：
  7/3（周五）买入 000757 → hold_days=1
  7/6（周一）管线运行 → hold_days=2（跳过周末）
  7/7（周二）管线运行 → hold_days=3
```

### 7.4 完整核算示例

```
初始资金 = 1,000,000

7/3 买入 000757，成本 49,940.47
  现金 = 1,000,000 - 49,940.47 = 950,059.53
  000757市值 = 12,900 × 3.8704 = 49,928.16
  总资产 = 950,059.53 + 49,928.16 = 999,987.69 ✅

7/6 买入 000858，成本 43,761.31
  现金 = 1,000,000 - 49,940.47 - 43,761.31 = 906,298.22
  000757市值 = 12,900 × 3.77 = 48,633.00
  000858市值 = 600 × 73.76 = 44,256.00
  总资产 = 906,298.22 + 48,633.00 + 44,256.00 = 999,187.22 ✅
  累计收益 = (999,187.22 / 1,000,000) - 1 = -0.08%（手续费）
```

## 八、数据库表结构

### 8.1 四张核心表

| 表名 | 记录 | 生命周期 | 关键字段 |
|:----|:-----|:---------|:---------|
| `sim_buy_signals` | 买入信号 | pending → executed / cancelled | symbol, llm_score, status, buy_date, cancel_reason |
| `sim_positions` | **当前持仓** | 买入→卖出前 | symbol, shares, total_cost, highest_price, pnl, hold_days, today_opened, pending_sell_reason, llm_override_count |
| `sim_closed_trades` | **已完成交易** | 卖出后永久保存 | symbol, buy/sell_date, pnl, sharpe_ratio, max_drawdown, exit_reason |
| `sim_account_daily` | 账户日结 | 每日一条 | cash, stock_value, total_value, total_return, position_count |

### 8.2 表间关系

```
sim_buy_signals                           sim_closed_trades
  │ symbol                                   │ symbol
  │ status (pending/executed/cancelled)       │ buy/sell_date
  │                                           │ pnl, sharpe, max_drawdown
  ▼                                           │ exit_reason
sim_positions                                 ▲
  │ symbol, shares, total_cost                │
  │ current_price, current_value               │
  │ pnl, pnl_pct (浮动盈亏)                   │
  │ hold_days, today_opened                    │
  │ pending_sell_reason → 非空=明天卖出        │
  │ llm_override_count                        │
  │                                           │
  └─ 卖出后移到 ──────────────────────────────┘

  sim_account_daily
  │ date
  │ cash = INITIAL_CAPITAL - Σ(pos_cost) + Σ(closed_pnl)
  │ stock_value = Σ(shares × current_price)
  │ total_value = cash + stock_value
```

## 九、异常场景汇总

| 场景 | 处理方式 | 信号状态 | 后续 |
|:-----|:---------|:---------|:------|
| 买入时涨停 | 取消信号 | `cancelled` | 今晚 LLM 可能重新推荐 |
| 买入时跌停 | 取消信号 | `cancelled` | 同上 |
| 买入时停牌 | 取消信号 | `cancelled` | 同上 |
| 买入时仓位满 | 取消多余信号 | `cancelled` | 卖出后会释放仓位，新推荐递补 |
| 买入时资金不够1手 | 取消信号（如贵州茅台） | `cancelled` | 预算5万不变，高价股自动放弃 |
| 买入时现金不足 | 取消信号 | `cancelled` | 卖出后回笼现金再买 |
| 卖出时跌停 | 取消待卖出标记 | `pending_sell`清空 | 今晚重新评分，可能再次标记 |
| 卖出时停牌 | 取消待卖出标记 | `pending_sell`清空 | 同上 |
| 评分触发+LLM推荐 | 覆盖卖出（≤3次） | 持仓不变，`llm_override_count`+1 | 连续3次后强制执行 |
| 评分触发+LLM推荐（≥3次） | 强制执行卖出 | `pending_sell`写入 | 次日开盘卖出 |
| 连续两日无行情数据 | 跳过评估 | 持仓不变 | 数据恢复后重新评估 |
| 非交易日（周末/节假日） | 整条管线跳过 | 不变 | 下一交易日恢复 |
