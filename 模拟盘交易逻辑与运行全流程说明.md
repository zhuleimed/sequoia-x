# Sequoia-X 模拟盘交易逻辑与运行全流程说明

> 版本: v1.0 | 更新: 2026-07-02
> 
> 本文档完整记录模拟盘从荐股到买卖、换股的全套逻辑，供系统维护和后续开发参考。

---

## 一、核心原则

| # | 原则 | 说明 |
|:--|:-----|:------|
| 1 | **T+1 模型** | 所有买卖决策基于 T-1 日收盘价数据，在 T 日以开盘价执行 |
| 2 | **先卖后买** | 每日执行顺序：先卖出（释放仓位）→ 再买入（递补） |
| 3 | **取消不顺延** | 涨停/跌停/停牌/仓位不足 → 取消信号，不留存过夜 |
| 4 | **LLM 覆盖三次上限** | 卖出规则触发时若 LLM 同日推荐，可覆盖 ≤ 3 次，之后强制执行 |
| 5 | **评分优先** | 买入信号按 LLM 评分降序排列，仓位不够时只取高分 |


## 二、时间线模型

### 2.1 交易日流程

```
           T-1 日                              T 日                          T+1 日
      ═══════════════                    ═══════════                    ═══════════════

      09:30-15:00 盘中交易              09:30-15:00 盘中交易           09:30-15:00 盘中交易
      15:00 CLOSE 收盘                  15:00 CLOSE 收盘               15:00 CLOSE 收盘

18:10 cron 触发管线                  18:10 cron 触发管线              18:10 cron 触发管线
  │                                     │                                │
  ├─ Step1: 数据同步                    ├─ Step1: 数据同步                ├─ Step1: 数据同步
  │  拉取 T-1 日行情入库                │  拉取 T 日行情入库              │  拉取 T+1 日行情入库
  │                                     │                                │
  ├─ Step2: LLM 推荐                    ├─ Step2: LLM 推荐               ├─ Step2: LLM 推荐
  │  写入 pending 信号                  │  写入 pending 信号              │  写入 pending 信号
  │                                     │                                │
  └─ Step3: --sim-update                └─ Step3: --sim-update           └─ Step3: --sim-update
      ├─ 执行待卖出(OPEN)                   ├─ 执行待卖出(OPEN)               ├─ 执行待卖出(OPEN)
      ├─ 执行待买入(OPEN)                   ├─ 执行待买入(OPEN)               ├─ 执行待买入(OPEN)
      └─ 评估 + 标记待卖出(CLOSE)           └─ 评估 + 标记待卖出(CLOSE)       └─ 评估 + 标记待卖出(CLOSE)
```

### 2.2 关键时序说明

```
T-1日 18:10 评估持仓（用 T-1 日 CLOSE）
  → 触发卖出条件 → 标记 pending_sell（不实际卖出）

T日 18:10 执行待卖出（用 T 日 OPEN）
  → pending_sell 标记的持仓以 OPEN 卖出 → 释放仓位
  → 再用 T 日 OPEN 买入 T-1 日 LLM 推荐的股票 → 递补
  → 评估持仓（用 T 日 CLOSE）→ 标记新的 pending_sell

T+1 日重复上述流程
```

| 术语 | 含义 |
|:----|:------|
| **CLOSE 价** | 用于评估（多因子评分），决定是否触发买卖条件 |
| **OPEN 价** | 用于执行（实际买卖操作），决定成交价格 |
| **T-1 日推荐** | LLM 基于 T-1 日完整行情数据的推荐，T 日执行买入 |
| **T 日推荐** | 当天 LLM 的推荐，写入 pending 信号，T+1 日执行 |


## 三、买入逻辑

### 3.1 信号来源

```
LLM 报告 → parse_llm_report()
  ├─ 解析 "RECOMMEND: 600519,000858" 行（最优先）
  ├─ 解析 "最优关注: ..." 行中的股票代码
  └─ 回退：按多策略频率取前 2 只
```

LLM 每次推荐 **不超过 2 只**。

### 3.2 信号写入 → 排序 → 执行

```
T-1日 Step2: LLM 推荐
  → insert_buy_signals_batch() 写入 sim_buy_signals
  → buy_date = T-1, status = 'pending', llm_score = N.N
  ↓
T-1日 Step3: 这些信号不被处理（buy_date = T-1 不满足 < T-1）
  ↓
T日 Step3: get_pending_signals() 
  → SELECT ... WHERE status='pending' AND buy_date < 'T'
  → 只取到 T-1 日及之前的信号
  → 按 llm_score DESC 降序排列

slots_available = MAX_POSITIONS(20) - 当前持仓数

signals = [...评分4.5..., ...评分4.0...]  ← 排序后
signals[:slots_available]  → 执行买入
signals[slots_available:]  → 取消（mark_signal_cancelled）
```

### 3.3 买入检查清单

每只信号依次检查以下条件，全部通过才执行买入：

| # | 检查 | 失败处理 |
|:--|:-----|:---------|
| 1 | 今日有行情数据 | 取消信号 |
| 2 | 未涨停（open < prev_close × 1.10） | 取消信号 |
| 3 | 未跌停（open > prev_close × 0.90） | 取消信号 |
| 4 | 未停牌（volume > 0） | 取消信号 |
| 5 | 资金足够（预算 5 万） | 重算或取消 |
| 6 | 仓位未满（＜ 20 只） | 取消 |

### 3.4 买入价格模型

```
买入价 = OPEN × (1 + SLIPPAGE)
SLIPPAGE = 万1（0.0001）

股数 = floor(预算 5万 / 买入价 / 100) × 100   ← 向下取整到手

成本 = 股数 × 买入价
佣金 = max(成本 × 万2.5, 0)
总成本 = 成本 + 佣金

剩余现金 = 现金 - 总成本
```

### 3.5 仓位不足处理

```
当前持仓 19 只，LLM 推荐 2 支（评分 4.5 和 4.0）
  ↓
slots_available = 20 - 19 = 1
signals[:1] → 评分 4.5 的执行买入 ✅
signals[1:] → 评分 4.0 的取消丢弃 ❌（不留存）
  ↓
下一交易日 LLM 重新推荐 → 新信号递补（不用旧信号）
```


## 四、卖出逻辑

### 四阶段流程

```
┌─────────────────────────────────────────────────────────┐
│  阶段一：估值更新（T-1 日 CLOSE）                        │
│  对所有持仓：更新 current_price / pnl / pnl_pct          │
│  更新 highest_price / hold_days / 当日估值              │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  阶段二：多因子评分（T-1 日 CLOSE）                      │
│  对每只持仓运行 evaluate_exit()                         │
│  13 条规则各自评分，累加总分                            │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  阶段三：决策（规则引擎 + LLM 覆盖）                    │
│  总分 ≥ 60 或 S1 硬止损穿透？                           │
│    ├─ 是 → LLM 同日推荐该股？                           │
│    │   ├─ 是且覆盖次数 < 3 → 跳过卖出，计数器+1        │
│    │   ├─ 是且覆盖次数 ≥ 3 → 强制执行卖出              │
│    │   └─ 否 → mark_position_for_sell()                 │
│    └─ 否 → 继续持有 ✅                                  │
└─────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────┐
│  阶段四：执行卖出（T 日 OPEN）                           │
│  检查跌停/停牌/无行情 → 取消标记                        │
│  全部通过 → 以 OPEN 价执行卖出                          │
│  计算净收入/佣金/印花税/PnL                             │
│  写入 sim_closed_trades                                  │
│  推送单股平仓报告                                        │
└─────────────────────────────────────────────────────────┘
```

### 4.1 多因子评分规则（13 条）

| 编号 | 规则 | 条件 | 分值 | 穿透？ |
|:----:|:-----|:------|:----:|:------:|
| S1 | 硬止损 | 亏损 > 8% | **100** | ✅ 穿透 |
| S2 | 止损预警 | 亏损 > 5% | 40 | |
| T1 | 移动止盈强卖 | 从最高回落 ≥ 8%（须先盈利 ≥ 15%） | **85** | |
| T2 | 移动止盈预警 | 从最高回落 ≥ 5%（须先盈利 ≥ 15%） | 50 | |
| D1 | 时间止损 | 持有 > 20 日 | **75** | |
| D2 | 时间预警 | 持有 > 15 日 | 40 | |
| M1 | 均线死叉确认 | MA5<MA10 连续 ≥ 3 日 | **70** | |
| M2 | 均线死叉初现 | MA5<MA10 单日 | 40 | |
| SH1 | 夏普恶化 | 15 日夏普 < -0.5 | **70** | |
| SH2 | 夏普转负 | 10 日夏普 < 0 | 50 | |
| SH3 | 夏普偏低 | 15 日夏普 < 0.5 | 30 | |
| R1 | 相对弱势 | 近 5 日跑输上证 > 5% | **60** | |
| R2 | 相对弱势预警 | 近 5 日跑输上证 > 3% | 30 | |

**触发条件**：总分 ≥ 60，或 S1 穿透（强制卖出）。

**S1 硬止损穿透逻辑**：一旦亏损 > 8%，直接卖出，不经过总分判断，也不经任何覆盖。

### 4.2 卖出价格模型

```
卖出价 = OPEN × (1 - SLIPPAGE)
收入 = 股数 × 卖出价
佣金 = max(收入 × 万2.5, 0)
印花税 = 收入 × 千1（仅卖出时收取）

净收入 = 收入 - 佣金 - 印花税
盈亏 = 净收入 - 持仓总成本
收益率 = 盈亏 / 持仓总成本
```

### 4.3 T+1 保护（today_opened）

| 场景 | today_opened | 卖出评分 |
|:-----|:------------:|:---------|
| T 日买入的股票，T 日评估 | 1 | 跳过（不触发卖出） |
| T+1 日评估 | 0 | 正常评分 |
| T+N 日评估 | 0 | 正常评分 |

### 4.4 失败处理（取消不顺延）

```
跌停（open ≤ prev_close × 0.90）→ clear_pending_sell()
  → 今晚 _update_and_evaluate() 重新评估
  → 若仍触发 → 再标记 → 明日重试（自然恢复机制）

停牌（volume = 0）→ clear_pending_sell()
  → 同上
```


## 五、LLM 覆盖机制

### 5.1 覆盖条件

```
卖出触发（评分 ≥ 60）
  AND 该股在今日 LLM 推荐中（buy_date = today, status = 'pending')
  → LLM 覆盖生效
```

### 5.2 计数器机制

```
sim_positions.llm_override_count（默认 0）
  └─ 每次覆盖 increment_llm_override() → +1

覆盖次数 < 3  → 跳过卖出，继续持有
覆盖次数 ≥ 3  → 不再覆盖，强制 mark_position_for_sell()
                  exit_reason="LLM覆盖3次后强制卖出(...)"
```

### 5.3 计数器重置

```
正常标记 pending_sell（未被覆盖）→ reset_llm_override() → 0
买入新股票 → 初始化 llm_override_count = 0
卖出后持仓删除 → 计数器自动废弃
```

### 5.4 设计理由

LLM 做全维度分析（大盘/板块/基本面/新闻），技术规则只看价格指标。LLM 说"买"可能看到了技术指标盲区。但连续 3 次都说"卖"仍被覆盖，说明趋势确实恶化，不应再扛。


## 六、先卖后买 + 仓位递补

### 6.1 执行顺序

```
run_daily() 中：
  Step 1: _execute_pending_orders()
    ├─ _execute_pending_sells()  ← 先执行待卖出（OPEN 价）
    │   → 释放仓位
    │   → 增加可用现金
    └─ _execute_pending_buys()    ← 再执行待买入（OPEN 价）
        → slots_available = 20 - 卖出后的持仓数
        → 递补买入

  Step 2: _update_and_evaluate()  ← 最后评估（CLOSE 价）
    → 更新估值
    → 运行卖出规则
    → 标记 pending_sell（明日执行）
```

### 6.2 仓位递补示例

```
T-1 日收盘：持仓 20 只（满仓）
  LLM 推荐 2 支（评分 4.5 和 4.0）→ 写入 pending 信号
  1 只触发卖出 → 标记 pending_sell

T 日：
  _execute_pending_sells() → 卖出 1 只（OPEN）→ 持仓 19 只
  _execute_pending_buys()  → 可用仓位 = 1
    → signals[:1] → 评分 4.5 的买入 ✅
    → signals[1:] → 评分 4.0 的取消 ❌
```


## 七、账户与报告体系

### 7.1 数据库表

| 表名 | 记录 | 关键数据 |
|:----|:-----|:---------|
| `sim_buy_signals` | 买入信号全生命周期 | pending→executed/cancelled, llm_score, buy_price |
| `sim_positions` | 当前持仓 | symbol, shares, cost, highest_price, pnl, hold_days |
| `sim_closed_trades` | 已完成交易（**全生命周期**） | buy/sell date, PnL, sharpe, max_drawdown, exit_reason |
| `sim_account_daily` | 每日账户快照 | cash, stock_value, total_value, total_return |

### 7.2 推送报告类型

| 报告类型 | 触发时机 | 内容 |
|:---------|:---------|:------|
| **单股平仓报告** | 每笔卖出时 | 交易明细、盈亏、夏普、退出原因 |
| **组合日报** | 每个交易日 | 账户概况、今日操作、当前持仓 |
| **月度汇总报告** | 每月末 | 月收益、全部交易清单、胜率/盈亏比/回撤 |


## 八、参数配置总览

全部参数集中在 `sequoia_x/simulation/config.py`，按功能分为 6 个区块：

| 区块 | 关键参数 | 默认值 |
|:-----|:---------|:-------|
| 账户 | `INITIAL_CAPITAL` / `PER_STOCK_BUDGET` / `MAX_POSITIONS` | 100万 / 5万 / 20只 |
| 交易成本 | `COMMISSION_RATE` / `STAMP_TAX_RATE` / `SLIPPAGE` | 万2.5 / 千1 / 万1 |
| 卖出阈值 | `SELL_THRESHOLD` | 60分 |
| 硬止损 | `HARD_STOP_LOSS` | -8%（穿透） |
| 移动止盈 | `TRAILING_ACTIVATE` / `TRAILING_T1` / `TRAILING_T2` | 15% / 8% / 5% |
| LLM 覆盖 | `increment_llm_override() 中 MAX_OVERRIDE` | 3次 |


## 九、异常场景汇总

| 场景 | 处理方式 | 信号状态 |
|:-----|:---------|:---------|
| 买入时涨停 | 取消信号 | `cancelled` |
| 买入时跌停 | 取消信号 | `cancelled` |
| 买入时停牌 | 取消信号 | `cancelled` |
| 买入时仓位满 | 取消多余信号 | `cancelled` |
| 买入时资金不足 | 取消信号 | `cancelled` |
| 卖出时跌停 | 取消待卖出标记 | `pending_sell_reason` 清空 |
| 卖出时停牌 | 取消待卖出标记 | `pending_sell_reason` 清空 |
| 评分触发+LLM推荐 | 覆盖卖出（≤3次） | 持仓不变 |
| 评分触发+LLM推荐（≥3次） | 强制执行卖出 | `pending_sell_reason` 写入 |
| 连续两日无行情数据 | 跳过评估，持仓不变 | 不变 |
