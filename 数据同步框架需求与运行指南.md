# Sequoia-X 数据同步框架需求与运行指南

> 版本: v2.6 | 最后更新: 2026-06-24

---

## 一、框架概述

### 1.1 目标

每个交易日自动将 **5206只A股** 和 **6个指数** 的日线数据完整拉取入库，确保：

- 所有未退市股票每个交易日都有记录
- 停牌股票保留前值价格 + volume=0
- 指数日线与股票同步更新
- 任意数据源故障时自动切换备用源

### 1.2 核心文件

| 文件 | 职责 |
|:----|:-----|
| `sequoia_x/data/sync.py` | DataSync 类：同步管线入口、增量拉取、缺失修复、估值回填、指数同步 |
| `sequoia_x/data/engine.py` | DataEngine 类：数据库初始化、基础查询、股票池过滤 |
| `sequoia_x/data/tencent_source.py` | TencentSource 类：腾讯+新浪双API数据源 |
| `fill_extra_fields.py` | 扩展字段补填脚本（amount/pctChg/peTTM等，一次性运行） |
| `main.py` | 命令行入口（--sync-only / --repair / 日常模式） |

### 1.3 核心原则

```
每只股票、每个交易日、必须有记录（退市除外）
停牌 ≠ 退市 → volume=0 + 价格沿用前值
数据源优先级: baostock(全字段) → Tencent(OHLCV) → 新浪(备选)
只有退市才能停止更新数据
```

## 二、数据源架构

### 2.1 多数据源切换机制

```
Baostock (主数据源)
  ├─ adjustflag="2" (前复权)
  ├─ 字段: date,open,high,low,close,volume,amount,turn,pctChg,peTTM,pbMRQ,psTTM,pcfNcfTTM
  ├─ 端口 10070，基于 TCP 长连接
  └─ 不稳定 — 有时超时或服务端主动断开

TencentSource (第一备用)
  ├─ web.ifzq.gtimg.cn (腾讯证券API，端口80)
  ├─ 前复权(参数qfq)
  ├─ 字段: date,open,high,low,close,volume(部分提供amount)
  ├─ peTTM/pbMRQ/psTTM/pcfNcfTTM = None(后续由估值回填阶段补充)
  ├─ 稳定，约0.34s/只，不受 baostock 宕机影响
  └─ 覆盖 6 大指数日线

SinaSource (第二备用，TencentSource内部实现)
  ├─ quotes.sina.cn
  └─ Tencent 失败时自动切换
```

### 2.2 多源切换流程

```python
# sync_daily 对每只股票:
if baostock_available:
    try baostock query
    if success → data = baostock_data (含全部字段)
    if failed  → baostock_available = False, fall through

if data is None:
    try TencentSource.get_daily()
    if success → data = tencent_data (peTTM等=None)
    if failed  → fall through

if data is None:
    consecutive_errors += 1, continue (跳过此股票)

_write_to_db(data)  # 写入缓冲区
```

### 2.3 baostock 可用性探测

```python
data_source: str = "baostock"              # "baostock" | "tencent"
tencent_count_since_switch: int = 0        # Tencent 累计处理数
TENCENT_BATCH_SIZE: int = 200              # 每 200 只尝试恢复 baostock

# 流程：
baostock 模式 → 使用 baostock 查询（含全部字段）
baostock 失败/异常 → data_source = "tencent"，切 Tencent
Tencent 模式 → tencent_count_since_switch += 1
tencent_count_since_switch >= 200 → 尝试一次 baostock 恢复
  → 成功 → data_source = "baostock"，当场用其数据
  → 失败 → 继续 Tencent
Tencent 降级数据不含估值字段，由 Phase 4 _fill_valuation_gaps 回填
```

### 2.4 指数数据源

```
sync_index_daily:
  1. 尝试 baostock → 成功则使用 baostock 数据（含全部字段）
  2. baostock 失败 → 自动切换 TencentSource.get_daily()（OHLCV）
  3. 双源均失败 → 跳过该指数，下个交易日重试
```

## 三、同步管线 (5 Phases)

### 3.1 完整管线

```
run_full()
  │
  ├─ Phase 1: sync_stock_list()
  │   同步股票列表，检测上市/退市
  │   代码: bs.query_stock_basic() → 对比本地 stock_list 表
  │   耗时: ~30秒
  │   关键: type="1"(股票) status="1"(上市) 双重过滤
  │
  ├─ Phase 1b: _archive_delisted_stocks()
  │   退市数据归档：将 stock_daily 中退市股数据移至 stock_daily_archive
  │   退市股仍保留在 stock_list（标记 delisted_date），但行情数据从主表清理
  │   确保 get_local_symbols()、策略计算和推送统计不再包含退市股
  │   耗时: 与退市股数量相关，一般 < 1s
  │
  ├─ Phase 2: sync_daily(force=False)
  │   增量日线同步（Tencent 主力，baostock 后备）
  │   交易日判断: is_trade_day() 3层策略
  │   第1层: check_date.weekday()>=5(周六/日) → 直接False，0ms
  │   第2层: baostock query_trade_dates → 精确含节假日
  │   第3层: chinese_calendar(is_workday) → baostock不可用时离线判断
  │   第4层: fail-open → 所有数据源失败则True(宁可多拉不可漏)
  │   数据源: Tencent(优先) → baostock(后备)
  │   字段: 每只股票拉取 OHLCV+成交量+成交额，计算 pctChg
  │   耗时: 25~40分钟(Tencent主力) / 40~60分钟(baostock后备)
  │   关键: requests_since_login 计数器，每 1400 次主动重连
  │         _write_to_db 保留停牌数据(volume=0)
  │
  ├─ Phase 2b: _fill_ohlcv_gaps(days=5)
  │   OHLCV 历史缺失补填（Tencent，INSERT OR IGNORE 不覆盖已有数据）
  │   确保技术指标计算所需连续 K 线，最多处理 200 只
  │   数据源: 仅 Tencent
  │   耗时: < 2分钟
  │
  ├─ Phase 3: _fill_valuation_gaps(days=5)
  │   baostock 估值字段补充（peTTM/pbMRQ/psTTM/pcfNcfTTM）
  │   策略暂未用到估值字段，baostock 能拉就拉，超 2h 跳过
  │   数据源: 仅 baostock（Tencent 不提供估值字段）
  │   关键: 与策略运行解耦，不影响管线主流程
  │
  └─ Phase 4: sync_index_daily()
      同步 6 大指数日线
      数据源: baostock(优先) → TencentSource(回退)
      关键: 不依赖 Phase 2/3 结果，独立执行
```

### 3.2 逐阶段详细说明

#### Phase 2: sync_daily

```python
# 1. 交易日判断
if not force and not self.is_trade_day():
    return {"status": "skipped", ...}

# 2. 增量范围计算
# 每只股票: start = last_date + 1, end = today
# 条件: start <= today_str 才拉取

# 3. 主动重连保护
requests_since_login >= 1400 → _proactive_reconnect()

# 4. 双轨拉取（Tencent 主力 → baostock 后备）
# Tencent: get_daily(5天) + get_realtime(成交额)
#   - OHLCV + 成交量 + 成交额
#   - pctChg 从 close 计算
#   - 估值字段设为 None（由 Phase 3 补充）
# baostock: 全字段含估值（Tencent 不可用时后备）
# 写入缓冲区

# 5. 批量写入
# 缓冲区满 500 条 → _flush_batch → SQLite executemany

# 6. 每 100 只进度日志
# 格式: "进度: N/5206 (有效写入 M) [来源:Tencent/baostock 本批100只 Xs]"
```

#### Phase 2b: _fill_ohlcv_gaps

```python
# 1. 调用 check_missing(days=5) 获取 OHLCV 缺失报告
# 2. 按缺失数量降序排列，最多处理 200 只
# 3. 对每只缺失股票用 Tencent 拉取 7 天数据
# 4. 计算 pctChg = (close - prev_close) / prev_close × 100
# 5. INSERT OR IGNORE 写入（不覆盖已有 baostock 全字段数据）
# 6. 耗时 < 2 分钟，确保策略所需 K 线连续
```

#### Phase 3: _fill_valuation_gaps

```python
# 1. 快速探测 baostock 可用性
#    query_trade_dates(start=today, end=today)
#    if error → 跳过(不影响管线)
# 2. 查询 stock_daily 中最近 N 天 peTTM IS NULL 的记录
# 3. 逐条用 baostock 回查(5s 超时保护)
# 4. UPDATE 覆盖原值
# 5. 策略暂未用到估值字段，超 2h 跳过
```

#### Phase 5: sync_index_daily

```python
# 6 大指数:
INDEX_CODES = {
    "sh.000001": "上证指数", "sh.000016": "上证50",
    "sh.000300": "沪深300",  "sh.000905": "中证500",
    "sz.399001": "深证成指", "sz.399106": "深证综指",
}
# 对每个指数:
#   1. 获取本地最新日期 → start_date = last_date + 1
#   2. 尝试 baostock → 成功则写入
#   3. baostock 失败 → TencentSource(指数代码与baostock相同)
#   4. 写入 index_daily 表(独立于 stock_daily)
```

### 3.3 管线错误传递

```
Phase 1 error → 终止管线，推送告警
Phase 2 error → 终止管线(数据不全，选股无意义)
Phase 2 skipped → 继续 Phase 3(非交易日本身不需要选股)
Phase 3 error → 不终止管线(估值字段暂未被策略使用)

Phase 4 error → 不终止管线(指数数据为辅助参考)
```

## 五、错误处理

### 5.1 故障保护

| 层次 | 触发条件 | 操作 | 耗时 |
|:-----|:--------|:-----|:----:|
| 主动重连 | requests_since_login >= 1400 | 提前登出再登录 | 0.3s |
| 数据源切换 | baostock 失败 | 自动切 TencentSource | 无额外耗时 |
| 周期性恢复 | Tencent 模式每 200 只 | 尝试一次 baostock，成功切回 | 单次查询 |
| 缺失补填 | 次日 Phase 3 | 重新拉取缺失区间 | 依赖数据量 |

### 5.2 baostock 常见错误码

| 错误码 | 含义 | 处理方式 |
|:------|:-----|:---------|
| 10002007 | 接收数据异常，请稍后再试 | 日志记录，切 Tencent |
| 10001001 | 系统繁忙 | 切 Tencent 回退 |
| Broken pipe | 服务端强制断开连接 | 主动重连机制介入 |

### 5.3 TencentSource 出错

TencentSource 内部已实现新浪API作为第二备用，调用 `TencentSource.get_daily()` 时：

1. 先请求腾讯 API (`web.ifzq.gtimg.cn`)
2. 腾讯失败 → 自动切换新浪 API (`quotes.sina.cn`)
3. 双源均失败 → 返回 None → sync.py 跳过此股票

## 八、常见问题

### Q1: 同步卡在 repair_missing 一小时无日志？

**已解决。** v2.2 新增了：
1. repair_missing 每 30 秒输出进度日志
2. baostock 失败时自动切 Tencent 回退(不会卡住)
3. _fill_valuation_gaps 进入前先探测 baostock，不可用则跳过

### Q2: 双轨模式每只股票都打日志，文件太大？

**已解决。** v2.2 将每只股票的 Tencent 回退日志从 INFO 降为 DEBUG，只有 Phase 切换和进度批次保留在 INFO 级别。

### Q3: baostock 登录失败后 repair_missing 就终止了？

**已解决。** v2.2 改为警告后继续执行，baostock 查询失败时自动切 Tencent 回退。

### Q4: 指数日线不同步？

**已解决。** v2.2 sync_index_daily 新增 Tencent 回退(原仅 baostock)，且独立于 Phase 2/3/4 结果。

### Q5: baostock 恢复后如何回填 Tencent 期间缺失的估值字段？

由 Phase 4 `_fill_valuation_gaps()` 自动处理。该阶段查询 stock_daily 中 peTTM 为空的记录，用 baostock 逐条回查并 UPDATE。

### Q6: 数据库中的 .db 文件还有哪些文件？

`data/` 目录下应只有 `sequoia_v2.db` 一个文件。旧版产生的 `ashare_daily.db` 和 `results/` 目录已清理。

### Q7: 停牌股票数据怎么处理？

停牌股票(换Tencent也拿不到当日数据)：
1. 查询最后一条记录的价格
2. 写入新记录：volume=0, amount=0, pctChg=0, 价格沿用前值
3. 绝不允许因停牌而"某只股票某天无记录"

## 九、全自动管线（pipeline）

> 版本: v1.0 | 引入日期: 2026-06-22

### 9.1 动机

数据同步耗时不可预测（18:10 启动，快则 40min、慢则 2h+），原固定 19:00 启动策略选股无法保证数据已就绪。改为**链式触发**：上一步完成→立即启动下一步，不依赖时钟。

### 9.2 架构

```
cron 唯一入口 18:10
  │
  └─ python pipeline/pipeline.py
       │
       ├─ Step 1: 数据同步 (main.py --sync-only)       必需 | 超时4h
       ├─ Step 2: 策略选股+LLM (main.py)                必需 | 超时1h
       ├─ Step 3: 018 LSTM 策略                       可选 | 超时30min
       ├─ Step 4: 018 指标策略                         可选 | 超时30min
       └─ ... 未来项目
```

### 9.3 核心文件

| 文件 | 职责 |
|:----|:-----|
| `pipeline/pipeline.py` | 主编排器：读取步骤配置、逐步骤执行、超时保护、状态写入 |
| `pipeline/status.py` | 状态文件读写 + WxPusher 全管线推送 |
